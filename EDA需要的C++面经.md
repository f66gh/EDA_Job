# 🟩 **模块 2：类与对象语法基础（当前重点）**



> 这一块是 C++ 的门面，学完这个你就能看懂 OpenROAD 的大部分头文件结构了。



### 2.1 struct / class & 访问控制



- 【⭐必会】`struct` vs `class` 的默认访问权限区别
- 【⭐必会】`public / protected / private` 语义
- 【★较重要】封装思想：**对外接口最少，对内细节隐藏**（EDA 软件复杂度高，封装不好不仅乱，还容易出 Bug）



### 2.2 成员函数、this 指针与 const 成员



- 【⭐必会】`this` 指针的本质
- 【⭐必会】`const` 成员函数：
  - **EDA 考点**：在一个 `const GetTiming()` 函数里，能不能修改缓存变量？（引出 `mutable`）
- 【★较重要】`mutable` 成员：用于逻辑上 const 但物理上需要修改的场景（如统计调用次数、Lazy Evaluation 缓存）。



### 2.3 静态成员 & 静态成员函数



- 【⭐必会】`static` 数据成员：类内声明，类外定义。
- 【⭐必会】`static` 成员函数：没有 `this` 指针。
- 【★较重要】**EDA 场景**：用 static 成员做全局唯一的 `NetID` 计数器，或者存储全局配置（如工艺角 Process Corner）。



### 2.4 构造 / 析构 基础



- 【⭐必会】构造顺序：**基类 → 成员 → 自己**
- 【⭐必会】析构顺序：**自己 → 成员 → 基类**
- 【★较重要】`explicit` 关键字：防止构造函数发生隐式类型转换（避免把一个 `int` 莫名其妙转成一个 `Node` 对象）。

# 🟧 **模块 3：STL 容器与现代语法（提至此处，结合刷题）**



> **调整理由**：你每天都在刷力扣，这部分结合你的实战经验最容易掌握。加入了 Modern C++ 必备语法。



### 3.1 现代 C++ 基础语法（✨ 新增）



- 【⭐必会】`auto` 关键字：
  - 不是为了偷懒，是为了解决复杂模板类型难写的问题（如迭代器类型）。
- 【⭐必会】Range-based for loop（范围 for 循环）：
  - `for (const auto& node : nodes) { ... }` —— EDA 遍历图的标准写法。
- 【⭐必会】Lambda 表达式：
  - **lambda 本质是一个对象**（这个对象的类型叫闭包类型）
  - `[capture](params){ body }`
  - **实战**：`std::sort` 自定义排序规则时必用（例如按 Net 的延时大小排序）。
- 函数指针和Lambda表达式的关系——只有不捕获任何东西的 lambda 才能隐式转换为函数指针
  - 在回调函数中的应用




### 3.2 顺序容器：vector / string / deque



- 【⭐必会】`std::vector`：
  - **EDA 核心**：为什么它是 EDA 最爱？（连续内存 = Cache Friendly）。
  - `reserve()` vs `resize()`：预分配内存避免 realloc 带来的性能开销。
- 【⭐必会】`std::string` vs `std::string_view`（✨ 新增 C++17）：
  - `string_view` 是只读窗口，**无拷贝**。解析超大 Verilog/LEF 文件时，性能比 `string` 高得多。



### 3.3 关联容器：map / unordered_map



- 【⭐必会】`std::unordered_map`：
  - 底层哈希表，查找 O(1)。用于 `NameString -> Node*` 的快速映射。
- 【⭐必会】`std::map`：
  - 底层红黑树，Key 有序。
- 【★较重要】自定义 Key：如果用结构体当 Key，`map` 需要重载 `<`，`unordered_map` 需要写 `hash` 函数。



### 3.4 常用算法



- 【⭐必会】`std::sort`（快排+堆排混合）、`std::reverse`
  - 迭代器（写 C++ 的面试很常被问）：
    - **随机访问（RandomAccessIterator）**
    - **双向（BidirectionalIterator）**
    - **前向（ForwardIterator）**

- 【★较重要】`std::lower_bound` / `std::upper_bound`：在有序数组中二分查找（STA 查表插值计算 Delay 时常用）。

# 🟦 **模块 4：对象内存布局 & 性能优化（进阶）**



> 学完容器后，再来看内存，你就明白为什么 `vector` 比 `list` 快了。



### 4.1 对齐与 Padding



- 【⭐必会】内存对齐规则 & Padding。
- 【★较重要】`sizeof` 计算：空类大小是 1（为了占位）。



### 4.2 Cache Friendly（缓存友好）（✨ 重点增强）



- 【⭐必会】**局部性原理**：
  - CPU 读内存是一次读一个 Cache Line（通常 64 字节）。
  - **面试金句**：为什么 STA 图节点要存数组里而不是散落在堆上的指针？**“为了提高空间局部性，减少 Cache Miss，提升遍历速度。”**
- 【★较重要】Structure of Arrays (SoA) vs Array of Structures (AoS)：EDA 大数据量下的布局优化策略。



### 4.3 多态与虚函数表



- 【⭐必会】虚函数表（vtable）与虚指针（vptr）。
- 【⭐必会】动态多态的开销：
  - 内存开销（每个对象多一个指针）。
  - 运行时开销（间接寻址 + 阻碍 inline）。
  - **EDA 观点**：在核心算法（如时序传播）的最内层循环，尽量避免虚函数调用。

# 🟥 **模块 5：拷贝控制 / 移动语义 / 智能指针**



> C++ 的分水岭。这里从“能写代码”进阶到“能写**安全且高效**的代码”。



### 5.1 深拷贝 vs 浅拷贝



- 【⭐必会】Rule of Five（五大函数）。
- 【⭐必会】什么时候需要深拷贝？（管理裸指针资源时）。



### 5.2 右值引用 & 移动语义（C++11）



- 【⭐必会】左值 vs 右值（纯右值/将亡值）。
- 【⭐必会】`std::move` 的本质：强制转为右值引用，不发生移动。
- 【★较重要】移动构造函数：**窃取资源**而不是拷贝资源（EDA 处理大图对象时，能移动绝不拷贝）。



### 5.3 智能指针（RAII）



- 【⭐必会】`std::unique_ptr`：独占资源，零开销。EDA 树形结构首选。
- 【⭐必会】`std::shared_ptr`：引用计数。**注意循环引用问题**（用 `weak_ptr` 解）。
- 【★较重要】**EDA 陷阱**：千万别对几亿个 Net 对象都用 `shared_ptr`，内存和引用计数更新的开销扛不住。
